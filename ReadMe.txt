МЕХАНИЗМ АВТОМАТИЗАЦИИ SQL-ЗАПРОСОВ
----------------------------------------------------------------------
Править SQL-код в "CreateDataBase.cs" и "CRUD.cs" вручную не нужно. Запросы автоматизированы через интерполяцию строк C#.

Принцип работы:
1. В SQL-запросах вместо названий таблиц указаны переменные:
   CREATE TABLE IF NOT EXISTS {Config.Table1Name}
2. При замене значения кавычек в файле "Config.cs":
   Table1Name = "Clients"; // вместо "Suppliers"
3. Программа при старте автоматически подставляет строку "Clients" во все команды CREATE, SELECT, INSERT и UPDATE.

Итог: Перестройка всей базы данных под новую тему происходит строго в одном файле "Config.cs".


Код интерфейса (MainWindow.xaml.cs) сам автоматически подхватит эти 
новые слова через динамическую интерполяцию полей. Заходить туда и 
что-то исправлять руками БОЛЬШЕ НЕ НАДО!

ШАГ 2: МЕНЯЕМ РУССКИЙ ТЕКСТ НА КНОПКАХ И ВКЛАДКАХ (MainWindow.xaml)

ШАГ 3: ИЗМЕНЕНИЕ ВИЗУАЛА С ПОМОЩЬЮ ОКНА «СВОЙСТВА» (БЕЗ КОДА)
----------------------------------------------------------------------
Если ты боишься случайно стереть кавычки или скобки в коде XAML, 
изменяй дизайн мышкой через специальную панель.

1. КАК ОТКРЫТЬ ОКНО СВОЙСТВ:
   * Открой файл "MainWindow.xaml".
   * Нажми на кнопку или таблицу прямо на визуальном экране конструктора.
   * Если справа снизу нет панели "Свойства" (Properties), нажми на 
     клавиатуре комбинацию: Alt + Enter (или F4).
   * Справа появится удобная таблица со всеми настройками этого элемента.

2. КАК КРАСИТЬ И МЕНЯТЬ ШРИФТЫ МЫШКОЙ:
   * Выдели нужную кнопку на экране.
   * В окне свойств найди вкладку "Кисти" (Brushes) или "Текст" (Text).
   * Чтобы изменить цвет кнопки: нажми на "Background" и выбери любой 
     цвет на палитре. Код сам допишется в нужную строчку!
   * Чтобы изменить цвет текста: нажми на "Foreground" и выбери цвет.
   * Во вкладке "Текст" можно в один клик сделать шрифт жирным (Bold) 
     или увеличить его размер (FontSize).

3. ЗОЛОТОЕ ПРАВИЛО ПРИ РАБОТЕ СО СВОЙСТВАМИ:
   Поменял цвет у одной кнопки — СРАЗУ запусти программу (F5) и проверь, 
   что ничего не зависло. Помни про Ctrl + Z, если мышка дрогнула! 
   И главное — НЕ ЗАКРЫВАЙ Visual Studio, если что-то сломалось, пока 
   не отменишь действие назад.

ШАГ 4: ПЕРЕНОС ЭЛЕМЕНТОВ ИНТЕРФЕЙСА КУРСОРОМ В КОНСТРУКТОРЕ (ПОДРОБНАЯ ИНСТРУКЦИЯ)
----------------------------------------------------------------------
Открой MainWindow.xaml в Visual Studio и переключись на вкладку «Дизайн» (внизу экрана). Появится окно программы с кнопками и таблицами.

Найди элемент, который хочешь переместить (кнопку, таблицу и т. д.), и кликни по нему левой кнопкой мыши. Вокруг него появится синяя рамка — значит, он выделен.

Зажми левую кнопку мыши на выделенном элементе и перетащи его туда, куда нужно. Во время перемещения появятся направляющие линии — они помогут ровно поставить элемент.

Когда элемент будет на нужном месте, отпусти кнопку мыши — он зафиксируется.

Сразу запусти программу — нажми F5. Проверь:

стоит ли элемент там, где ты его поставил;

не перекрывает ли он другие элементы;

нормально ли всё выглядит, если изменить размер окна;

работают ли кнопки и таблицы как раньше.

Если что‑то пошло не так, не закрывай Visual Studio. Жми Ctrl + Z (отмена действия), пока всё не вернётся назад.

ШАГ 5: ЗОЛОТЫЕ ПРАВИЛА РАЗРАБОТКИ ДЛЯ ТЕХ, КТО В ТАНКЕ
----------------------------------------------------------------------
Чувак, если ты не хочешь в панике переписывать весь проект за 5 минут 
до конца экзамена, зазубри два главных правила:

1. ПРАВИЛО «ОДНО ИЗМЕНЕНИЕ — ОДИН ЗАПУСК»
   Сделал одну правку (например, перекрасил одну кнопку или перенес одну

   НИКОГДА не меняй всё скопом! Если ты перепишешь 20 строк подряд, а

2. МАГИЧЕСКАЯ КОМБИНАЦИЯ (Ctrl + Z)
   Если ты что-то удалил, стер кавычку, скобку, или после твоей правки 
   всё покрылось красными линиями и перестало запускаться — НЕ ПАНИКУЙ.
   Просто нажми клавиатурную комбинацию Ctrl + Z (отмена действия). 
   Нажимай её несколько раз, пока код не вернется в то состояние, когда 
   всё работало. Это твой главный спасательный круг.

    ВНИМАНИЕ! КРИТИЧЕСКИЙ МОМЕНТ:
   История Ctrl + Z работает ТОЛЬКО пока открыта Visual Studio! Если ты 
   накосячил в коде, у тебя всё сломалось, и ты со страху ЗАКРЫЛ Visual Studio 
   или перезагрузил комп — ВСЁ, Ctrl + Z больше НЕ ПОМОЖЕТ. История правок 
   стерется навсегда, и тебе придется чинить код вручную. 
   Поэтому: накосячил -> НЕ закрываешь студию -> жмешь Ctrl + Z пока не починится!

ШАГ 6: КАК ЭТО ВООБЩЕ РАБОТАЕТ (ИНСТРУКЦИЯ ДЛЯ ОПЕРАТОРА)
----------------------------------------------------------------------
Если на показе препод или эксперт скажет: «Ну давай, продемонстрируй 
работу системы», не тупи. Вот как устроено управление:

1. ГЛАВНЫЕ КНОПКИ (Сверху или сбоку, куда ты их перенес):
   * Нажатие на них просто создает шаблонную запись "по дефолту" 
     в соответствующую таблицу. Нажал «Добавить контрагента» — в таблице 
     появилась строчка.

2. МАГИЯ ПРАВОЙ КНОПКИ МЫШИ (ПКМ) — КОНТЕКСТНОЕ МЕНЮ:
   Вся основная работа с базой скрыта внутри самих таблиц (DataGrid).
   * Выдели мышкой любую строчку в таблице.
   * Нажми на неё ПРАВОЙ кнопкой мыши (ПКМ).
   * Выскочит секретное контекстное меню. В нем есть три действия:
     - «Добавить» — открывает окно создания.
     - «Изменить» — открывает красивое модальное окно. Там ты можешь 
       стереть стандартный текст и написать реальные данные (например, 
       название реальной Аптеки или ФИО врача). Нажимаешь «Сохранить» — 
       и данные в таблице обновляются!
     - «Удалить» — полностью стирает эту строчку из базы данных.

3. ДВОЙНОЙ КЛИК (Быстрое редактирование):
   Если лень тыкать ПКМ, просто кликни два раза левой кнопкой мыши (ЛКМ) 
   по любой строчке с поставщиком или товаром — программа сама откроет 
   окно для изменения данных.

4. СВЯЗЬ ТАБЛИЦ (Как смотреть состав закупки):
   Перейди на вкладку «Реестр закупок» (Заказы). Нажми один раз на любую 
   строчку сделки. В этот момент нижняя таблица «Спецификация» автоматически 
   обновится и покажет только те лекарства/товары, которые привязаны 
   именно к этой закупке.

 ШАГ 8: ДВА ЖЕСТКИХ ПРАВИЛА ПЕРЕД САМЫМ ЗАПУСКОМ (ФИНАЛ)
----------------------------------------------------------------------
Все изменения внесены, визуал настроен. Теперь 
осталось выполнить два механических действия, без которых программа 
просто не заведется на компьютерах экзаменационной комиссии:

ПРАВИЛО №1: ПЕРЕКЛЮЧЕНИЕ СБОРКИ В РЕЖИМ x86 (ОБЯЗАТЕЛЬНО НА ДЕМЭКЗАМЕНЕ!)
По умолчанию Visual Studio может запускать проект в режиме "Any CPU" или x64. На компьютерах экспертов из-за этого может намертво зависнуть библиотека Microsoft.Data.Sqlite, так как ей нужна четкая разрядность.
* Посмотри на самую верхнюю панель в Visual Studio (там, где находится зеленая стрелочка запуска "Пуск").
* Рядом со стрелочкой найди выпадающий список, где написано "Any CPU" или "x64".
* Нажми на него и строго выбери режим **x86** (или зайди в Диспетчер конфигураций и создай платформу x86 для активной сборки).
* Сделай пересборку проекта: Сборка -> Пересобрать решение.
* Запускай! Теперь программа скомпилирована в стабильном 32-битном режиме, который без проблем запустится на любом старом железе и пройдет любую проверку комиссии.

(если версия visual studio доисторическая: открывай не решение а папку проекта и выбирай пункт запуска WpfApp.csproj)

Как объяснить выбор x86 экспертам:
"Целевая сборка приложения была принудительно переведена в архитектуру x86 
для обеспечения гарантированной кроссплатформенной совместимости с 
32-битными нативными компонентами движка СУБД SQLite, а также для 
исключения конфликтов несовпадения разрядности адресного пространства (BadImageFormatException) при развертывании ИС на целевых рабочих местах экспертной комиссии".
